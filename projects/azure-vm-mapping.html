Azure VM Mapping Tool

Python Azure API Pandas Excel

Maps on-prem VM inventory to Azure SKUs using the Azure Resource API to estimate monthly cloud migration costs across multiple pricing tiers.

Problem

Cloud migration planning requires mapping hundreds of on-premises VMs to the right Azure SKUs and estimating costs across different pricing models (Pay-As-You-Go, 1-Year Reserved, 3-Year Reserved, with and without Azure Hybrid Benefit). Doing this manually per VM is error-prone and slow.

Approach

  • Inventory Input: Reads CSV/Excel files with on-prem VM specs — device name, core count, total RAM, disk size, OS
  • Azure Pricing API: Fetches real-time VM pricing from the Azure Swagger pricing calculator API
  • Mapping Logic: For each customer VM, finds Azure SKUs matching cores and RAM thresholds (85% cores, 65% RAM for dev; production uses tighter thresholds)
  • SKU Selection: From matching VMs, selects the lowest disk size first, then lowest price
  • Pricing Tiers: Outputs costs for PAYGO, 1-Year Reserved, 3-Year Reserved — each with AHUB (Azure Hybrid Use Benefit) variants
  • Export: Generates Excel files per environment: Dev_Pricing_Output_{customer}_{region}.xlsx and Prod_Pricing_Output_{customer}_{region}.xlsx

Technical Details

  • API client class (GetData) handles Azure pricing API calls and response parsing
  • VM name parser extracts size, OS, and tier from Azure naming conventions (handles 3–5 part names)
  • Separate mapping functions for dev vs. prod with different resource utilization targets
  • Validation ensures exactly one Azure VM match per input VM
  • Supports both CSV and Excel input formats

Impact

  • Reduces migration cost estimation from days to minutes for large inventories
  • Consistent, repeatable pricing across environments and regions
  • Enables quick comparison across pricing tiers for budget planning
  • Excel output ready for stakeholder review without additional formatting
