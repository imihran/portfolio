Job Hunt Template

Python SQLite CLI Automation Shell Data Modeling

Local-first toolkit for tracking recruiter outreach, role pitches, compensation ranges, and follow-ups without putting private job-search data into git.

Problem

Recruiter outreach quickly becomes a messy mix of email threads, role descriptions, compensation ranges, status updates, and follow-up reminders. A spreadsheet works at first, but it does not preserve clean relationships between recruiters, agencies, opportunities, compensation data, and next actions.

Approach

  • Local storage: Uses SQLite so the working database stays private and portable
  • Structured schema: Separates recruiter records from opportunities, statuses, compensation, and source metadata
  • Idempotent upserts: Accepts JSON input and updates recruiter or opportunity rows without duplicating records
  • Compensation parsing: Normalizes common hourly and annual compensation formats for easier comparison
  • Follow-up reporting: Prints due and overdue recruiter follow-ups from the command line
  • Reusable templates: Provides generic reply templates and a private persona example for customization

Technical Details

  • Python scripts operate on a local SQLite database created from schema.sql
  • JSON input makes the upsert helper easy to call from shell scripts, email tools, or AI-assisted workflows
  • .gitignore excludes the local database, private persona file, scrape output, and environment files
  • Optional mail digest script integrates with himalaya for users who want local email scanning
  • LinkedIn helper scripts are kept optional and write generated private source material outside version control

Impact

  • Turns recruiter outreach into a queryable local system instead of scattered inbox memory
  • Keeps sensitive job-search details out of public repos by design
  • Creates a reusable foundation for automating follow-ups, compensation tracking, and opportunity comparison
