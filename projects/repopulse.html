RepoPulse

Python LangGraph Postgres pgvector OpenAI API Cloudflare Pages

Autonomous repo-health monitoring system that watches GitHub activity, detects unusual changes, and writes cited root-cause reports for open-source maintainers.

Problem

Most GitHub analytics tools report metrics, but they do not explain why activity changed. Maintainers still have to inspect PRs, releases, issues, comments, and timing patterns by hand when merge times slow down or issue volume spikes.

Approach

  • Event ingestion: Downloads GH Archive data and filters it into Postgres for watched repositories
  • Metric detection: Computes daily repo-health metrics and flags anomalies with a MAD z-score baseline
  • Evidence enrichment: Pulls PR, issue, comment, and release context from the GitHub API
  • Semantic search: Chunks enriched artifacts, embeds them, and indexes them in pgvector for cited retrieval
  • Agent investigation: Uses a LangGraph loop to choose SQL, semantic search, or release-note tools based on the question
  • Static publishing: Exports reports to a Cloudflare Pages frontend with report details and citations

Technical Details

  • Postgres 16 with pgvector stores raw events, metrics, anomalies, artifacts, embeddings, reports, and citations
  • HNSW vector index supports fast approximate nearest-neighbor retrieval over PR and issue text
  • Pydantic validates the agent output contract so reports cannot include uncited claims
  • Guardrail test suite covers SQL safety, prompt injection resistance, and report grounding behavior
  • Makefile-driven local workflow runs Docker, ingestion, detection, enrichment, embedding, evaluation, and export commands

Impact

  • Turns repo-health monitoring from dashboards into incident-style explanations with evidence
  • Keeps investigation cost low with GPT-4o-mini and a bounded 6-tool-call agent budget
  • Demonstrates production-oriented AI engineering: retrieval, structured outputs, guardrails, observability, and evals
