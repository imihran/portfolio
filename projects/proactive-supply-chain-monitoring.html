Proactive Supply Chain Monitoring System

SQL ETL Statistical Analysis Alerting Systems Data Pipeline

Transformed vendor relationship management from reactive firefighting to proactive issue identification by building a multi-layered alarm system that saves ~3,640 hours annually across 70+ managers.

Problem

Supply chain managers had no early warning system for vendor issues. Problems were discovered 2-3 days after they occurred, and each manager spent ~1 hour/week manually checking for anomalies across their vendor portfolio. No prior solution existed in this space.

Approach

Designed a 4-alarm escalation workflow where each layer provides increasing specificity: vendor-level flagging → historical trend diagnosis → item-level root cause → inventory health monitoring. Built a 708-line SQL pipeline consolidating 9 data sources into a unified view.

Technical Details

  • 4-alarm escalation architecture: vendor-level triage → trend diagnosis → item root cause → inventory health
  • Statistical threshold logic using trailing 12-week/15-week min-max bands with fallback rules
  • Item-level anomaly detection using 60%/140% of trailing average thresholds
  • Weeks-of-Cover calculation consolidating 9 data sources (inventory, demand forecasts, buyability, procurability, attribution)
  • Color-coded alerting (RED/YELLOW) with automated daily scheduling (Mon-Fri)
  • Dual report variants: internal and external (vendor-facing)
  • Self-service query interface for next-week order plan visibility

Impact

  • ~3,640 hours saved annually (1 hr/week × 70 managers × 52 weeks)
  • Alerts delivered within 2-3 hours vs 2-3 days previously
  • Directly addresses vendor feedback about lack of early warning systems
  • Established new operational paradigm adopted across the supply chain organization
